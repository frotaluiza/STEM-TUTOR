### 3. Key Architectural Patterns

The architecture is an **agent-native** design with a **two-layer plugin model**:

```
Entry Points:  CLI (Typer)  |  WebSocket /api/v1/ws  |  Python SDK
                    ↓                    ↓                    ↓
              ┌──────────────────────────────────────────────────┐
              │              ChatOrchestrator                    │
              │   routes UnifiedContext → selected Capability    │
              └──────────┬──────────────┬────────────────────────┘
                         │              │
              ┌──────────▼──┐    ┌──────▼───────────┐
              │ ToolRegistry │    │ CapabilityRegistry│
              │  (Level 1)   │    │   (Level 2)       │
              └──────────────┘    └───────────────────┘
```

**Level 1 -- Tools (single-shot LLM-invoked functions):**
- User-toggleable: `brainstorm`, `web_search`, `paper_search`, `reason`
- Context-gated (auto-mounted): `rag`, `read_source`, `read_memory`, `write_memory`, `exec`, `code_execution`, `web_fetch`, `github`, `cron`, `ask_user`, `list_notebook`, `write_note`, `geogebra_analysis` (coming soon)
- Vision tools: `vision/image_utils`, `vision/block_parser`, `vision/coord_transform`, `vision/ggb_validator`

**Level 2 -- Capabilities (multi-stage pipelines that own a turn):**

| Capability | Stages |
|---|---|
| `chat` | exploring -> responding (default, single agentic loop) |
| `mastery_path` | responding (Guided Learning with mastery tools gated per topic type) |
| `auto` | analyzing -> delegating -> synthesizing (routes to another capability) |
| `deep_solve` | planning -> reasoning -> writing |
| `deep_question` | ideation -> generation |
| `deep_research` | rephrasing -> decomposing -> researching -> reporting |
| `visualize` | analyzing -> generating -> reviewing (SVG/Chart.js/Mermaid/HTML or Manim) |
| `math_animator` | concept_analysis -> concept_design -> code_generation -> code_retry -> summary -> render_output |

**Paper/Academic Search:**

**RAG / Knowledge Bases:**

**Memory System (3-Layer):**
- L1: Event traces (`trace/<surface>/<date>.jsonl`)
